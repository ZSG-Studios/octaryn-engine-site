# Contributing to Octaryn Engine

Thanks for helping.

This repository is access-controlled for verified contributors, but it is not
open source. Read these files before contributing:

- `licences/LICENSE.md`
- `licences/CONTRIBUTOR_ASSIGNMENT.md`
- `licences/CONTRIBUTOR_INTAKE.md`

## The short version

Verified contributors may:

- access the repository;
- clone or download it;
- build and test locally;
- make changes locally; and
- submit pull requests or patches back to the official Octaryn repositories.

You may not:

- redistribute the source;
- publish forks or mirrors;
- redistribute patches outside the official upstream workflow;
- distribute binaries, builds, or executables;
- reuse any code, assets, docs, tools, editor code, or engine code elsewhere;
- use the project commercially;
- upload the project to another host, store, launcher, archive, or package
  registry; or
- use any project material for AI training, datasets, benchmarking, or model
  improvement.

## Required intake before your first substantive PR

Before your first substantive pull request, you must complete the official
Octaryn Engine contributor intake flow.

That intake flow is where you:

- verify the GitHub account you will use for pull requests;
- acknowledge the project license restrictions;
- agree that only Zachary David Rose may distribute or commercialize Octaryn Engine unless separately authorized in writing; and
- acknowledge that submitted contributions are governed by `licences/CONTRIBUTOR_ASSIGNMENT.md`.

The live intake flow uses GitHub OAuth through a private Apps Script web app and
stores contributor details privately in a Google Sheet stored in Google Drive,
and in the private intake backup repo.

Verified intake submissions are reviewed by the maintainer before any private
repo collaborator invite is sent.

Current intake page URL:

`https://zsg-studios.github.io/octaryn-engine-site/contribute.html`

The intake flow verifies GitHub identity first and then may collect identity
details such as your legal name and email address privately in a Google Sheet
stored in Google Drive, and in the private intake backup repo.

Pull requests from contributors who have not completed that process, or whose
access has not been approved yet, may be
closed without review.

## Asset rule

All assets and non-code materials in this repository are covered by the same
license restrictions as the code.

Do not extract, reuse, remix, redistribute, or republish any artwork, textures,
models, meshes, animations, music, sound effects, writing, lore, logos,
branding, or other non-code materials from this repository.

## Contribution rule

By intentionally submitting a contribution, you confirm that:

- you have the legal right to submit it;
- your contribution does not knowingly include unauthorized third-party code,
  assets, or confidential material;
- you are submitting it only for review and possible inclusion in this project;
- you understand that only Zachary David Rose may publish, distribute, release,
  sublicense, or commercialize the project unless he separately authorizes
  otherwise in writing; and
- you have completed the contributor intake process referenced in
  `licences/CONTRIBUTOR_INTAKE.md` and understand that accepted submissions are governed
  by `licences/CONTRIBUTOR_ASSIGNMENT.md`.

## Pull request rules

- Keep PRs focused.
- Explain what changed and why.
- Do not include third-party code or assets unless clearly disclosed.
- Do not post public mirrors of your branch or change set.
- Do not attach compiled builds for public distribution.
- Do not rename or remove ownership, copyright, trademark, or license notices.
- Expect maintainers to ask for confirmation that your intake form is on file.

## Review and merge

All submissions are reviewed at maintainer discretion.
A contribution may be edited, rejected, requested for revision, merged in part,
merged with changes, or removed later.

## No expectation of release

A pull request being merged does not create any obligation to credit, release,
ship, or compensate.
